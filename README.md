# CPM2 Plates Market

Telegram-бот-маркетплейс виртуальных игровых номеров для Car Parking Multiplayer 2. Все номера — цифровые игровые активы: они не являются, не заменяют и не имитируют реальные государственные регистрационные знаки.

Стек намеренно минимальный: Python 3.13+, aiogram long polling, asyncpg и Railway PostgreSQL. Нет ORM, Alembic, Redis, S3, отдельной веб-админки или production Docker Compose.

## Возможности

- Валидация RU (строгая кириллица и справочник кодов регионов) и KZ (два формата, ASCII, регионы 01–20, чёрный список серий).
- Lazy minting: номер резервируется на пять минут, а после Telegram Stars оплаты передаётся игроку.
- Фиксированные продажи с настраиваемой комиссией; аукционы с заморозкой ставки, возвратом перебитой ставки и anti-sniping.
- Postgres FSM (`user_sessions`) и single-message UI — навигация редактирует одно сообщение интерфейса.
- Полностью встроенная админка: пользователи, блокировки, баланс, номера, аукционы, настройки, KZ blacklist, CMS-карточки и резервные копии.
- APScheduler: резервы, аукционы, уведомления, неактивные аккаунты и ежедневный `pg_dump` владельцу бота.

## Локальный запуск

1. Установите Python 3.13+ и PostgreSQL с утилитой `pg_dump`.
2. Создайте базу данных `cpm2_plates`.
3. Скопируйте `.env.example` в `.env` и заполните `BOT_TOKEN`, `DATABASE_URL`, `OWNER_TELEGRAM_ID`, `ADMIN_IDS`.
4. Установите зависимости и запустите бот:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
python -m app.main
```

При первом старте SQL-файлы из `migrations/` применяются по имени и фиксируются в `schema_migrations`. Не редактируйте уже применённые миграции: создавайте следующую пронумерованную `.sql`-миграцию.

Проверка:

```powershell
python -m pytest -q
```

## Деплой на Railway

1. Создайте новый Railway project.
2. Добавьте сервис **PostgreSQL**.
3. Добавьте сервис из этого репозитория. Railway использует приложенный `Dockerfile`.
4. Свяжите приложение с PostgreSQL: Railway подставит `DATABASE_URL`.
5. В Variables приложения задайте:

| Variable | Значение |
| --- | --- |
| `BOT_TOKEN` | Токен от BotFather |
| `DATABASE_URL` | Ссылка Railway PostgreSQL |
| `OWNER_TELEGRAM_ID` | Числовой Telegram ID владельца |
| `ADMIN_IDS` | Дополнительные ID через запятую, если нужны |
| `ENV` | `production` |
| `PORT` | `8080` (или Railway `PORT`) |

6. Deploy. При старте бот открывает пул PostgreSQL, выполняет миграции, запускает scheduler, `/health` и aiogram polling.

Railway health check уже описан в `railway.json` (`GET /health`). Никакие дополнительные Railway-сервисы не требуются.

## Экономика и защита целостности

- Все значения Stars — целые числа.
- Цена фиксированной продажи: ⭐1–⭐99 999.
- Аукцион: до 24 часов; минимальный шаг, anti-sniping и комиссия меняются в админке.
- Изменения владельца, балансов, резервов и ставок выполняются транзакционно через asyncpg. Изменяемые строки блокируются `SELECT ... FOR UPDATE`; для ещё не существующего номера lazy minting также берёт PostgreSQL advisory lock.
- `plates.plate_number` имеет вечный unique index. Даже возвращённый государству номер не может быть повторно эмитирован как новый.

## Администрирование

Откройте `/admin` с ID из `OWNER_TELEGRAM_ID`, `ADMIN_IDS` или таблицы `admins`. Интерфейс позволяет:

- смотреть статистику и финансовый свод;
- искать пользователей, блокировать/разблокировать их и корректировать баланс;
- принудительно передавать номер игроку или государству;
- принудительно завершать/отменять аукционы;
- менять платформенные настройки и KZ blacklist;
- редактировать CMS-карточки. Формат: `card_id|заголовок|описание`; для картинки отправьте фото с этой подписью — сохраняется Telegram `file_id`;
- запросить резервную копию PostgreSQL. Автоматическая копия отправляется владельцу каждый день.

## Переменные окружения

`BOT_TOKEN`, `DATABASE_URL`, `OWNER_TELEGRAM_ID` обязательны. `ADMIN_IDS`, `ENV` и `PORT` имеют значения по умолчанию, показанные в `.env.example`. Никогда не коммитьте `.env` или рабочие токены.
